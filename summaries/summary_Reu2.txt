**Resumen:**

**Tema:** Laboratorios de Chrome, S3, CloudFront, RDS y Landa

**Puntos Principales:**

1. Se llevó a cabo un laboratorio utilizando Chrome con la versión 3, que permite almacenar objetos y publicar páginas web estáticas en 200 puntos de conexión mediante CloudFront.
2. Se creó un bucket en S3 para alojar los archivos de la página web estática y se personalizaron las páginas de error para manejar errores en el enlace, URL o bloqueos geográficos.
3. Se configuró CloudFront para distribuir la página web estática y gestionar la seguridad mediante Routic53.
4. Se realizó un segundo laboratorio con RDS y Landa, creando una base de datos y una instancia para conectarse a los datos públicamente.
5. Se utilizó Amazon RDSM para insertar y consultar datos en la base de datos, además de crear una función de Python para conectarse al RDSM y mostrar los datos.

**Detalles Importantes:**

- Se personalizaron las páginas de error en S3 para gestionar errores en el enlace, URL o bloqueos geográficos.
- Se creó un grupo de seguridad para conectarse al puerto de la base de datos en Landa y RDS.
- Se insertaron datos en la base de datos utilizando Amazon RDSM y se creó una función de Python para conectarse al RDSM.

**Conclusiones o Acciones:**

- CloudFront mejora la atención y destruye mejor la página web estática en 200 puntos de conexión.
- Las páginas de error personalizadas en S3 permiten gestionar errores en el enlace, URL o bloqueos geográficos.
- Landa y RDS permiten crear una base de datos y una instancia para gestionar los datos públicamente.
- La creación de una función de Python permite insertar y consultar datos en la base de datos.
