**Resumen:**

**Tema:** Prácticas de laboratorio con AWS (Amazon Web Services)

**Puntos Principales:**

1. Se llevó a cabo un laboratorio utilizando Chrome y CloudFront para distribuir una página web estática en 200 puntos de conexión, protegiéndola con una web firewall y un servicio de DNS (Route 53).
2. Se explicó la creación de buckets y carpetas en Amazon S3, incluyendo una carpeta para páginas de error personalizadas.
3. Se presentó un segundo laboratorio utilizando RDS y landa para crear una base de datos con una instancia, conectándose a ella públicamente a través de Warbench o landa.
4. Se ilustró la creación de una tabla de estudiantes y la inserción de datos en la base de datos.
5. Se desarrolló una función de Python para conectarse al RDSM y mostrar los datos en el MiceCole Warbench.

**Detalles Importantes:**

- La utilización de Chrome y CloudFront mejora la atención y distribución en diferentes puntos de conexión.
- Se protege la página web con una web firewall y un servicio de DNS (Route 53), asumiendo un costo por este servicio.
- Se crearon buckets y carpetas en Amazon S3, incluyendo una carpeta para páginas de error personalizadas, las cuales son manejadas por CloudFront.
- Se desarrolló una función de Python para conectarse al RDSM y mostrar los datos en el MiceCole Warbench.

**Conclusiones o Acciones:**

- La combinación de Chrome y CloudFront resulta beneficiosa para la distribución y atención de páginas web estáticas.
- La protección de páginas web a través de servicios adicionales es crucial para salvaguardar la integridad y seguridad del sitio.
- El desarrollo de funciones personalizadas en Python puede facilitar la manipulación y visualización de datos en bases de datos.
- Es importante tener en cuenta los costos asociados a los servicios adicionales utilizados en AWS.
